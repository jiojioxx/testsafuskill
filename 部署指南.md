# GoPlus ClawHub 部署指南

## 架构说明

本项目采用**前后端分离开发，生产环境统一部署**的架构：

- **开发环境**：前端（React）和后端（NestJS）独立启动，提升开发体验
- **生产环境**：NestJS 服务前端静态文件，只需部署一个服务

## 开发环境

### 1. 启动数据库

```bash
docker-compose up db -d
```

### 2. 启动后端（开发模式）

```bash
# 在项目根目录
npm run backend

# 或者进入 backend 目录
cd packages/backend
npm run dev
```

后端将运行在 `http://localhost:3000`，API 路径为 `/api/*`

### 3. 启动前端（开发模式）

```bash
# 在项目根目录
npm run frontend

# 或者进入 frontend 目录
cd packages/frontend
npm run dev
```

前端将运行在 `http://localhost:5173`

### 4. 开发环境特点

- ✅ 前端热更新，修改代码立即生效
- ✅ 后端自动重启，支持调试
- ✅ CORS 已配置，前后端可以正常通信
- ✅ 前端访问 `http://localhost:5173`
- ✅ API 请求发送到 `http://localhost:3000/api/*`

## 生产环境部署

### 方案一：Docker Compose 部署（推荐）

#### 1. 配置环境变量

创建 `.env` 文件：

```bash
# GitHub OAuth（可选）
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret

# 数据库密码（建议修改）
MYSQL_ROOT_PASSWORD=your_secure_root_password
MYSQL_PASSWORD=your_secure_password

# JWT 密钥（必须修改）
JWT_SECRET=your_super_secure_jwt_secret_key
```

#### 2. 构建并启动服务

```bash
# 构建镜像并启动所有服务
docker-compose up -d --build

# 查看日志
docker-compose logs -f app

# 停止服务
docker-compose down

# 停止并删除数据
docker-compose down -v
```

#### 3. 数据库迁移

```bash
# 进入容器执行迁移
docker-compose exec app npx prisma migrate deploy --schema=./packages/backend/prisma/schema.prisma
```

#### 4. 访问应用

打开浏览器访问：`http://localhost:3000`

- 前端页面：`http://localhost:3000`
- API 接口：`http://localhost:3000/api/*`

---

### 方案二：手动部署

#### 1. 构建前端

```bash
cd packages/frontend
npm install
npm run build
```

构建产物在 `packages/frontend/dist` 目录

#### 2. 构建后端

```bash
cd packages/backend
npm install
npm run build
```

构建产物在 `packages/backend/dist` 目录

#### 3. 配置环境变量

创建 `.env` 文件：

```bash
NODE_ENV=production
DATABASE_URL=mysql://user:password@localhost:3306/clawhub
JWT_SECRET=your_jwt_secret
UPLOAD_DIR=/app/storage/uploads
MAX_FILE_SIZE=52428800
```

#### 4. 启动应用

```bash
cd packages/backend
node dist/main.js
```

应用将运行在 `http://localhost:3000`

## 生产环境架构说明

### 请求流程

```
用户浏览器
    │
    ▼
http://localhost:3000
    │
    ├─ /api/*  ────────► NestJS API 处理
    │
    └─ /*      ────────► 返回前端静态文件 (index.html)
```

### 目录结构（生产环境）

```
/app/
├── packages/
│   ├── backend/
│   │   └── dist/           # 后端编译产物
│   └── frontend/
│       └── dist/           # 前端编译产物
│           ├── index.html
│           ├── assets/
│           └── ...
└── storage/
    └── uploads/            # 文件上传目录
```

### 静态文件服务逻辑

后端 `main.ts` 中的关键代码：

```typescript
// 生产环境：服务前端静态文件
if (process.env.NODE_ENV === 'production') {
  const frontendPath = join(__dirname, '..', '..', 'frontend', 'dist');
  
  // 静态资源服务
  app.useStaticAssets(frontendPath, {
    prefix: '/',
    index: false,
  });

  // 所有非 API 路由返回 index.html（支持前端路由）
  app.use((req, res, next) => {
    if (!req.path.startsWith('/api')) {
      res.sendFile(join(frontendPath, 'index.html'));
    } else {
      next();
    }
  });
}
```

## 常见问题

### 1. 前端页面 404

**原因**：前端构建产物路径不正确

**解决**：
```bash
# 检查前端构建产物是否存在
ls packages/frontend/dist

# 重新构建前端
cd packages/frontend && npm run build
```

### 2. API 请求失败

**原因**：环境变量未正确设置

**解决**：
```bash
# 检查环境变量
docker-compose exec app env | grep NODE_ENV

# 确保 NODE_ENV=production
```

### 3. 数据库连接失败

**原因**：数据库未启动或连接配置错误

**解决**：
```bash
# 检查数据库状态
docker-compose ps db

# 查看数据库日志
docker-compose logs db

# 测试数据库连接
docker-compose exec db mysql -u clawhub -p
```

### 4. 文件上传失败

**原因**：存储目录权限问题

**解决**：
```bash
# 检查存储目录
docker-compose exec app ls -la /app/storage

# 修复权限
docker-compose exec app chown -R nestjs:nodejs /app/storage
```

## 性能优化建议

### 1. 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name clawhub.example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf)$ {
        proxy_pass http://localhost:3000;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 2. 启用 HTTPS

```bash
# 使用 Let's Encrypt
sudo certbot --nginx -d clawhub.example.com
```

### 3. 数据库优化

```sql
-- 添加索引
CREATE INDEX idx_skills_download_count ON skills(download_count DESC);
CREATE INDEX idx_skills_created_at ON skills(created_at DESC);
```

## 备份和恢复

### 数据库备份

```bash
# 备份数据库
docker-compose exec db mysqldump -u clawhub -p clawhub > backup.sql

# 恢复数据库
docker-compose exec -T db mysql -u clawhub -p clawhub < backup.sql
```

### 文件备份

```bash
# 备份上传文件
docker-compose exec app tar -czf /tmp/uploads.tar.gz /app/storage/uploads

# 复制到本地
docker cp $(docker-compose ps -q app):/tmp/uploads.tar.gz ./uploads-backup.tar.gz
```

## 总结

这个部署方案的优势：

✅ **开发体验好**：前后端独立开发，热更新支持  
✅ **部署简单**：生产环境只需部署一个服务  
✅ **资源节省**：无需额外的 Nginx 或静态文件服务器  
✅ **维护方便**：统一的日志和监控  
✅ **扩展灵活**：后续可以轻松拆分为独立服务
